## 2.1 典故溯源

2002 年洛阳东周王城遗址出土的 **"天子驾六"车马坑**，以实物证实了这个先秦礼制：

```
六匹骏马并驾齐驱，拖载天子玉辂
  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐
  │左服│ │右服│ │左骖│ │右骖│ │左騑│ │右騑│
  │    │ │    │ │    │ │    │ │    │ │    │
  └───┘ └───┘ └───┘ └───┘ └───┘ └───┘
  └───── 天子玉辂 (帝王座驾) ──────┘
```

六匹骏马的排列暗合"六部"格局：**两骖左右、两服居中、两騑出前**

### 六马职责映射表

| 马位 | Dynasty 六部 | 古代礼制释义 | AI 职能 |
|------|-------------|-------------|---------|
| **左服** | 户部 (hubu) | 服马居辕内，承辂之重 | 数据核算 · 资源评估 |
| **右服** | 兵部 (bingbu) | 服马居中正，主驾之稳 | 代码研发 · Bug 修复 |
| **左骖** | 礼部 (libu) | 骖马在服外，辅以文德 | 文档规范 · 知识管理 |
| **右骖** | 刑部 (xingbu) | 骖马在服外，辅以威仪 | 安全审计 · 合规检查 |
| **左騑** | 吏部 (libu_hr) | 騑马行最外，开道斥候 | 注册考核 · 勋赏晋级 |
| **右騑** | 工部 (gongbu) | 騑马行最外，护辂清障 | CI/CD · 部署运维 |

## 2.3 驾六协同流转图

```
皇帝下旨 (执缰)
  │
太子承启 (理辔)
  │
┌─────────────────────────────────────────┐
│           三省审议 (校车)                │
│  中书草拟 → 门下审议 → 尚书派发          │
└─────────────────────────────────────────┘
  │         │         │         │         │         │
  ▼         ▼         ▼         ▼         ▼         ▼
户部核算   兵部研发   礼部规范   刑部审查   工部部署   吏部考核
(左服)    (右服)   (左骖)   (右骖)   (右騑)   (左騑)
  │         │         │         │         │         │
  └─────────┴─────────┴─────────┴─────────┴─────────┘
                          │
                     尚书汇总 (合辔)
                          │
                     早朝回奏 (归厩)
```

### 同步与异步模式

```
同步 (齐头并进):
  六部同时收到勒令 → 各有专属领域 → 并行执行

异步 (前后接应):
  户部核算数据 → 兵部基于数据进行开发
  礼部输出规范 → 工部按规范部署
  刑部审计完毕 → 吏部记录考核

阻塞恢复:
  任一马失蹄 → 天子勒缰 → 尚书重新调度
  对应: task blocked → emperor notified → shangshu reassigns
```

## 2.4 协同事件 API

```typescript
// 六马协同事件
export const ChariotEvents = {
  /** 天子勒缰 (暂停所有并行任务) */
  REIN_IN:       "chariot:rein-in",
  /** 天子策马 (加速推进) */
  SPUR_ON:       "chariot:spur-on",
  /** 马失前蹄 (任务阻塞) */
  STUMBLE:       "chariot:stumble",
  /** 尚书合辔 (汇总六部结果) */
  GATHER_REINS:  "chariot:gather-reins",
  /** 归厩 (任务完成归档) */
  RETURN_STABLE: "chariot:return-stable",
} as const;
```
