# 0103-YYC3-根基架构-圣旨消息载体

> **版本**：v1.0.0 ｜ **日期**：2026.06.14 ｜ **来源**：Dynasty Framework 第一章 1.3
>
> 圣旨卷轴是全链路消息的可视化载体，承载所有类型的政务通信。

---

## 1.1 圣旨类型体系

| 圣旨类型 | 系统消息类型 | 适用场景 | 流转路径 | 优先级 |
|----------|-------------|----------|----------|--------|
| **制书** | `edict_sys` | 全局制度变更、架构调整、项目启动（最高等级） | 皇帝→太子→三省→六部（全流程） | P0 - 最高 |
| **敕书** | `edict_norm` | 日常任务、常规需求（最常用） | 皇帝→太子→中书→门下→尚书→六部 | P1 - 高 |
| **敕牒** | `edict_fast` | 快速指令、临时调整、紧急修复 | 太子→中书→尚书→六部（门下事后补审） | P2 - 中 |
| **堂帖** | `edict_dept` | 尚书省内部调度、六部协同 | 尚书省→六部之间内部流转 | P3 - 普通 |
| **密奏/封事** | `memorial_secret` | 异常上报、风险预警，直达皇帝 | 六部/门下→直达天堂 | P0 - 紧急 |

## 1.2 圣旨四阶形态演变

```
┌─────────────────────────────────────────────────────┐
│                   圣旨形态生命周期                     │
│                                                     │
│  [初稿]      [草拟]       [审议]       [生效]        │
│   白麻   →   黄纸    →   绯绫    →   金泥           │
│                                                     │
│  空白圣旨   中书省印     门下省印     玉玺钤印        │
│  (无印章)   (朱红)       (朱红)       (金色)         │
│                                                     │
│  状态:      状态:        状态:        状态:          │
│  待承旨     待审议       待派发       已生效         │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## 1.3 数据结构定义

```typescript
interface EdictMessage {
  id: string;
  type: 'edict_sys' | 'edict_norm' | 'edict_fast' | 'edict_dept' | 'memorial_secret';
  title: string;
  content: string;
  
  // 流转状态
  stage: EdictStage;
  stageHistory: StageTransition[];
  
  // 印章记录
  seals: SealRecord[];
  
  // 元数据
  priority: 'P0' | 'P1' | 'P2' | 'P3';
  createdAt: Date;
  updatedAt: Date;
  creator: AgentRole;
}

type EdictStage = 
  | 'draft'       // 初稿 - 白麻
  | 'proposed'    // 草拟 - 黄纸
  | 'reviewed'    // 审议 - 绯绫
  | 'sealed'      // 生效 - 金泥
  | 'executing'   // 执行中
  | 'completed';  // 已办结
```
