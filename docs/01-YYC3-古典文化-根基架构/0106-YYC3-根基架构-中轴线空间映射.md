# 0106-YYC3-根基架构-中轴线空间映射

> **版本**：v1.0.0 ｜ **日期**：2026.06.14 ｜ **来源**：Dynasty Framework 第一章 1.6
>
> 看板的主流程，正好对应隋唐洛阳城的南北中轴线。流程节点直接"安"到真实的宫城建筑上。

---

## 1.1 地标—角色映射表

| 角色/流程 | 对应洛阳地标 | 空间寓意 | UI呈现建议 |
| :--- | :--- | :--- | :--- |
| **皇帝 (Emperor)** | **天堂 (礼佛堂)** | 权力与精神的制高点，发布最高旨意 | 看板最北端，金色/帝王紫高亮 |
| **太子 (Taizi)** | **明堂 (万象神宫)** | 处理、整理、发布政令的朝堂 | 皇帝下方，同轴紧挨 |
| **三省** | **应天门 (宫城正门)** | 政令审核、通过、正式发布的关口 | "三出阙"做视觉分隔，代表审核关卡 |
| **六部** | **天津桥 (中轴线桥梁)** | 政令分发给各部门的枢纽 | 天街两侧分布六部Agent |
| **早朝官 (Zaochao)** | **端门 (皇城正门)** | 每日定时播报、例行检查的固定节点 | 天津桥与定鼎门之间 |
| **回奏/汇总** | **定鼎门 (外郭城正门)** | 任务完成，最终成果与数据汇总的终点 | 看板最南端，代表任务闭环 |

---

## 1.2 三维映射图

```
        北（权力源头）
        │
   ┌────┴────┐  ── 天堂（天子）       → 降旨发令 / 全局最终决策 / 不可驳回
   │         │    数据：待御批数 / 全局健康度 / 密奏列表
   │  明堂   │  ── 明堂（储君）       → 旨意分拣 / 格式标准化 / 监国代行
   │         │    数据：待分拣数 / 当日旨意总量 / 监国状态
   ├─────────┤  ── 应天门（三省）     → 草拟·审议·派发 三权分立
   │中 门 尚 │    左阙中书: 待草拟 / 方案库匹配
   │         │    中阙门下: 待审议 / 典章合规检查
   ├──┬──┬──┤    右阙尚书: 待派发 / 令牌配额
   │户│礼│吏│
   │ 天津桥 │  ── 天津桥（六部）     → 令牌驱动执行 / 资源消费
   │兵│刑│工│    六部各自: 当前任务数 / 剩余令牌 / 运行状态
   ├──┴──┴──┤
   │  端门   │  ── 端门（早朝）       → 定时播报 / 健康巡检 / 风险预警
   │         │    数据：今日简报 / 异常事件 / 部门排行
   │ 定鼎门  │  ── 定鼎门（归档）     → 任务闭环 / 数据汇总 / 历史回溯
   │         │    数据：办结率 / 平均耗时 / 通过率 / 异常数
   └────┬────┘
         │
        南（执行终端）
```

## 1.3 数据流向规则

| 流向 | 方向 | 载体 | 触发条件 |
|------|------|------|----------|
| **下行传旨** | 北→南 | 圣旨卷轴（随阶段变色展开） | 上级节点操作完毕 |
| **上行回奏** | 南→北 | 折子文册（合订呈送） | 六部办结 / 尚书汇总 |
| **封驳回流** | 门下→中书（南→北回退） | 红色圣旨 + 驳议标签 | 门下审议不通过 |
| **密奏直达** | 六部→天堂（跳跃） | 加密封事 | 紧急风险 / 安全漏洞 |
| **令牌派发** | 尚书→六部（横向发放） | 铜质鱼符 | 任务经三省审批后 |
| **令牌缴回** | 六部→尚书（反向回收） | 鱼符飞回动画 | 任务办结 |
