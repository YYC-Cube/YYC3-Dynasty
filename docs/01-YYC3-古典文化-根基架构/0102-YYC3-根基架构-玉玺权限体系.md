# 0102-YYC3-根基架构-玉玺权限体系

> **版本**：v1.0.0 ｜ **日期**：2026.06.14 ｜ **来源**：Dynasty Framework 第一章 1.2
>
> 秦汉至隋唐形成"天子六玺"制度，不同玉玺对应不同政务场景，并非一枚玉玺管所有事务。

---

## 1.1 历史原型

> *"传国玉玺，正统象征，受命于天，既寿永昌。"*

2000 年来中国历史上最高权力的物质象征——**传国玉玺**，由和氏璧雕琢而成，方圆四寸，上纽交五龙。

---

## 1.2 天子六玺权限矩阵

| 玉玺名称 | 权限等级 | 对应系统功能 | 触发节点 |
|----------|---------|-------------|----------|
| **传国玉玺** | 最高级 | 全局最终决策签章、系统级配置修改、Agent生命周期管理、紧急熔断重置。**规则：加盖此玺的指令不可驳回、不可撤销，直接生效。** | 皇帝·天堂节点 |
| **皇帝行玺** | 政务级 | 日常行政任务生效签章、普通审批流程最终确认，绝大多数正式圣旨加盖此玺。 | 皇帝·日常决策 |
| **皇帝信玺** | 军事级 | 紧急任务、高优先级Bug修复、安全攻防类指令的授权签章，可跳过部分常规审批。 | 兵部/刑部 紧急任务 |
| **皇帝之玺** | 人事级 | Agent注册、权限变更、考核奖惩、岗位调动指令的专用签章。 | 吏部·人事任免 |
| **天子之玺** | 对外级 | 第三方系统对接、外部API调用、跨域资源申请的授权签章。 | 工部/对外联通 |

## 1.3 UI规则与交互规范

### 视觉表现

- 玉玺为立体朱红印章效果，印泥肌理叠加正片叠底（`--texture-yinni`）
- 未盖玺的圣旨显示"未生效"水印，六部可拒绝执行
- 封驳回退的圣旨，玉玺印自动变淡失效，重新审批后再次生效

### 权限校验流程

```
用户点击「钤印」
    │
    ▼
检查当前用户角色是否为天子/L4以上
    │
    ├─ 否 → 提示"无权使用此玺"，操作终止
    │
    ▼ 是
选择玉玺类型（根据操作场景自动推荐）
    │
    ▼
确认钤印（二次确认弹窗）
    │
    ├─ 取消 → 操作取消
    │
    ▼ 确认
执行钤印动画（玉玺落下 → 印泥扩散 → 印章显现）
    │
    ▼
更新圣旨状态为"已生效"
广播事件: 'seal:applied' → 全局同步
```
