## 8.1 RBAC 角色权限矩阵

### 权限层级定义

| 层级 | 角色 | 权限范围 | 可执行操作 |
|------|------|---------|-----------|
| **L0 - 访客** | 未登录 | 只读浏览 | 查看欢迎页、公开内容 |
| **L1 - 朝臣** | 普通用户 | 个人操作 | 创建自定义技能(受限)、查看个人勋章 |

## 8.2 并发控制与锁机制

### 乐观锁实现（低冲突场景）

```typescript
// 基于版本号的乐观锁
interface VersionedData {
  id: string;
  version: number;    // 版本号，每次更新递增
  data: any;
}

// 更新时携带预期版本，服务端校验
async function updateWithOptimisticLock(
  id: string,
  updater: (data: any) => any,
  expectedVersion: number
): Promise<Result> {
  const latest = await api.get(`/data/${id}`);
  
  if (latest.version !== expectedVersion) {
    return { success: false, conflict: true };  // 版本冲突
  }
  
  const newData = updater(latest.data);
  return await api.put(`/data/${id}`, {
    data: newData,
    expectedVersion
  });
}
```

### 悲观锁实现（高冲突场景）

```typescript
// 敕令编辑使用悲观锁
async function withPessimisticLock<T>(
  resourceId: string,
  operation: () => Promise<T>,
  lockTimeoutMs: number = 30000
): Promise<T> {
  // 1. 获取锁
  const lock = await api.post('/locks/acquire', {
    resource: resourceId,
    ttl: lockTimeoutMs
  });
  
  if (!lock.acquired) {
    throw new Error(`资源 ${resourceId} 正在被 ${lock.holder} 编辑`);
  }
  
  try {
    // 2. 执行业务操作
    const result = await operation();
    
    // 3. 释放锁
    await api.post('/locks/release', { lockId: lock.id });
    return result;
  } catch (error) {
    // 4. 异常时也要释放锁
    await api.post('/locks/release', { lockId: lock.id }).catch(() => {});
    throw error;
  }
}
```

### 适用场景对比

| 特性 | 乐观锁 | 悲观锁 |
|------|--------|--------|
| 冲突频率 | 低 | 高 |
| 性能影响 | 无锁开销 | 有锁等待 |
| 适用场景 | 个人设置、技能配置 | 敕令编辑、并发修改 |
| 实现复杂度 | 低 | 中 |
| 用户体验 | 冲突后提示合并 | 编辑前锁定提示 |
