## 7.1 全局数据流拓扑

### 主流程：对话→任务→流转→归档

```
P6 Hub浮窗 --[推送敕令]--> P5旨意板 --[七阶流转]--> 定鼎门归档
       |                      |                                  |
       |                      +--[办结触发]--> P4勋章墙(解锁检查)│
       |                                                        │
       +--[切换人格]--> P2朝堂大厅(节点高亮)                    │
```

### 辅流程：文化赋能→技能协同

```
P3时间轴 --[选择朝代]--> L1技能激活
     |                      
     +--[L1 Buff]---------> P5旨意板(敕令质量加成)
     |                      
     +--[创建技能]--------> 技能库更新 --> 全局广播
```

### 监控流：全局状态同步

```
P8太史监候 <--[轮询/WebSocket]-- 全页面状态聚合
     |                                                           
     +--[异常检测]--> P6 Hub浮窗(告警通知)                       |
     +->[健康指标]--> P2朝堂大厅(仪表盘更新)                     |
```

### 核心数据实体流转表

| 数据实体 | 产生源 | 流转路径 | 消费方 | 持久化 |
|---------|-------|---------|--------|--------|
| 敕令 (Edict) | P6 Hub / P5新建 | Hub→P5→七阶引擎→定鼎门 | P4勋章/P8监控 | ✅ edicts.ts |
| 技能状态 (Skill) | P3时间轴 | P3→全局广播→P2/P5/P8 | 全页面 | ✅ skills.ts |
| 勋章进度 (Honor) | 系统计算 | P5办结→触发检查→P4展示 | P4勋章墙 | ✅ honors.ts |
| 朝代选择 (Dynasty) | P3时间轴 | P3→全局广播→全页面 | L1Buff消费者 | 🔄 会话级 |
| 资源状态 (Resource) | P3/P5操作 | 变更→全局广播→P2/P8 | UI展示 | 🔄 内存+定时持久 |

## 7.2 事件驱动架构

### 事件生命周期（4阶段）

```
[发布] → [广播] → [处理] → [清理]
   │        │         │         │
eventBus  遍历订阅者 执行回调   自动解绑
.emit()  requestAnim  try-catch useEffect
          Frame()              cleanup
```

### 事件优先级队列

| 优先级 | 事件类型 | 示例 | 处理方式 |
|--------|---------|------|----------|
| **P0 - 实时** | 用户交互、状态变更 | `skill:activated`, `edict:created` | `requestAnimationFrame` |
| **P1 - 高** | 业务流程推进 | `edict:stage-changed`, `honor:unlocked` | `setTimeout(0)` |
| **P2 - 普通** | 数据同步、日志 | `resource:updated`, `metric:snapshot` | `requestIdleCallback` |
| **P3 - 低** | 分析统计、上报 | `analytics:page-view`, `log:batch` | 防抖 300ms |

### 事件总线实现要点

```typescript
// 核心原则：
// 1. 松耦合 - 页面间通过事件通信，不直接引用组件
// 2. 异步非阻塞 - 事件发布不等待处理完成
// 3. 错误隔离 - 单个处理器失败不影响其他处理器
// 4. 可追溯 - 所有事件记录日志，支持调试回放
```

## 7.3 跨页面状态同步机制

### 同步策略矩阵

| 数据类型 | 同步方式 | 一致性保证 | 冲突解决 |
|---------|---------|-----------|---------|
| 用户操作 | 事件广播 | 最终一致性 | 最后写入胜出 |
| 系统状态 | 单一数据源 | 强一致 | -- |
| 缓存数据 | 轮询+版本号 | 最终一致性 | 版本号大者胜出 |
| 实时指标 | WebSocket推送 | 最终一致性 | 服务端权威 |
