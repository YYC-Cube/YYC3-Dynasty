## 1.2 核心验收清单

### A. 文化根基验收 (时间+空间)

- [ ] 四大权力载体（玉玺/圣旨/竹箴/鱼符）均有技术落点，无纯装饰元素
- [ ] 中轴线8个地标节点完整映射到UI组件
- [ ] 十三王朝序列准确无误，26项技能全部定义
- [ ] 天子驾六模型六马→六部映射正确
- [ ] 所有古文化引用均有历史依据（标注出处）

### B. 制度架构验收 (空间+属性)

- [ ] 三省六部层级关系正确实现
- [ ] 七阶流转流程可完整执行
- [ ] 封驳回流机制正常工作
- [ ] 密奏直达通道权限控制正确
- [ ] 四道硬校验规则全部生效

### C. UI实现验收 (属性+事件)

- [ ] 9页核心页面全部按规格实现
- [ ] Design Tokens 100%覆盖，无硬编码色值
- [ ] Component + Variants + Auto Layout 规范遵循
- [ ] 所有状态覆盖（默认/悬浮/激活/禁用/异常/空状态）
- [ ] 动效时长符合规范（微交互<200ms, 过渡<500ms）
- [ ] 响应式断点适配完成

### D. 技术落地验收 (关联)

- [ ] TypeScript严格模式无编译错误
- [ ] 单元测试覆盖率 ≥ 80%
- [ ] E2E测试关键流程通过
- [ ] 性能指标达标（LCP<2.5s, FID<100ms, CLS<0.1）
- [ ] 无障碍合规（WCAG 2.1 AA级）

## 1.3 质量门禁标准

| 门禁项 | 通过标准 | 阻断级别 |
|--------|----------|----------|
| 编译错误 | 0 errors, ≤5 warnings | **阻断** |
| 测试覆盖率 | ≥80% (核心模块≥90%) | **阻断** |
| 类型安全 | `strict: true`, no `any` | **阻断** |
| Design Tokens 使用率 | ≥95% | 警告 |
| 组件化率 | 可复用组件≥80% | 警告 |
| 文档完整度 | API文档100%, 组件文档≥90% | 警告 |
