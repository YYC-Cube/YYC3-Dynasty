# 0501-YYC3-技术落地-Figma模型矩阵

> **版本**：v1.0.0 ｜ **日期**：2026.06.14 ｜ **来源**：Dynasty.md 第四章 4.1
>
> 基于 Figma Make 平台官方支持的 5 大 AI 模型，制定模型选型策略。

---

## 1.1 模型能力总览

| 模型 | 版本 | 核心特征 | 适用场景 | Credit消耗 | 推理风格 |
|------|------|----------|----------|------------|----------|
| **Claude Sonnet** | 4.6 | 多面手，效率与能力的最佳平衡 | 日常设计迭代、组件生成 | 标准 | 均衡型 |
| **Claude Opus** | 4.7 | 最强深度研究，挑战性项目 | 高精度视觉还原、复杂仪表盘 | 显著更高 | 深度透彻型 |
| **Gemini Flash** | 3 | 快速迭代，敏捷开发 | 快速原型探索、简单组件生成 | 较低 | 快速循环型 |
| **Gemini Pro** | 3.1 | 深度创意工作 | 复杂创意设计、动画SVG生成 | 中等偏高 | 深度创意型 |
| **GPT-5.5** | — | 能力全面，快速迭代 | 全栈设计任务、代码集成 | 中等 | 全面敏捷型 |

## 1.2 五阶段模型映射

```
┌─────────────────────────────────────────────────────────────┐
│                Dynasty Framework 设计生命周期                 │
│                                                             │
│  探索期 ──→ 设计期 ──→ 精修期 ──→ 动效期 ──→ 工程期         │
│                                                            │
│  Gemini    Claude      Claude     Gemini      GPT          │
│  3 Flash   Sonnet 4.6  Opus 4.7   3.1 Pro     5.5          │
│                                                            │
│  快速发散  均衡构建    像素精修    创意动效    全栈落地       │
│  低成本    标准成本    高成本      中高成本    中等成本       │
└─────────────────────────────────────────────────────────────┘
```

### 阶段一：概念探索（推荐 Gemini 3 Flash）

| 设计任务 | 选择理由 | 提示词策略要点 |
|----------|----------|----------------|
| P1 欢迎页初始布局探索 | 快速生成多个变体供筛选 | 强调"生成3个候选方案"，快速迭代 |
| 配色方案初步尝试 | 多色方案并行发散 | 提供"古风色板参考"，让AI发散 |
| 组件草图快速产出 | 简单组件秒级响应 | 单组件单提示词，批量生产 |

### 阶段二：核心页面精细（推荐 Claude Sonnet 4.6）

| 设计任务 | 选择理由 | 提示词策略要点 |
|----------|----------|----------------|
| P2 朝堂大厅中轴线布局 | 1M token上下文，可处理完整规格 | 一次性输入全部节点规格表 |
| Design Tokens 定义 | 编码能力强，命名规范统一 | 提供完整色彩/字体/间距表格 |
| Auto Layout 约束搭建 | 对布局引擎理解深入 | 明确父子层级、对齐方式 |

### 阶段三：高精度视觉（推荐 Claude Opus 4.7）

| 设计任务 | 选择理由 | 提示词策略要点 |
|----------|----------|----------------|
| P2 天子驾六仪表盘 | 3.75MP视觉升级，捕捉微妙细节 | 强调6马×3状态的像素级差异化 |
| 古风材质纹理渲染 | 最高分辨率视觉理解 | 详细描述宣纸/竹简/印泥肌理层次 |
| 全局设计一致性审查 | 深度审查能力，发现跨页不一致 | 要求对比9页原型的风格偏差 |

### 阶段四：创意动效（推荐 Gemini 3.1 Pro）

| 设计任务 | 选择理由 | 提示词策略要点 |
|----------|----------|----------------|
| 圣旨四阶变形动画 | 代码级动画生成，SVG原生支持 | 描述颜色渐变的节奏与触发条件 |
| 玉玺钤印启动动效 | 物理效果模拟出色 | 描述重力加速→接触→印泥扩散过程 |
| 勋章解锁发光特效 | 粒子/光效创意强 | 描述发光色温、扩散形状、衰减曲线 |

### 阶段五：全栈落地（推荐 GPT-5.5）

| 设计任务 | 选择理由 | 提示词策略要点 |
|----------|----------|----------------|
| Figma → React Code 生成 | Omnimodal原生架构，转换精准 | 同时提供截图和规范，双向约束 |
| Design Tokens → CSS变量导出 | 代码整洁度高，输出可直接使用 | 要求输出tokens.css + Tailwind config |
| 状态管理架构（七阶流转） | 擅长复杂状态机设计 | 描述完整状态转移矩阵和副作用 |

## 1.3 切换策略

- 连续2次输出不符合预期 → 切换至下一强度等级模型
- 设计阶段发生质的变化（如从布局→动效）→ 按阶段推荐切换
- Credit预算紧张但仍有质量要求 → Sonnet 4.6为最佳性价比
- 需要跨模型验证同一设计 → 用 Opus 4.7 做 final review
