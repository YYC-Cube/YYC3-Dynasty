# 0302-YYC3-流转机制-封驳校验规则

> **版本**：v1.0.0 ｜ **日期**：2026.06.14 ｜ **来源**：Dynasty Framework 第三章 3.2-3.4
>
> 封驳回流机制确保决策质量，四道硬校验保障执行合法性。

---

## 4.1 封驳回流（逆向闭环）

```
门下审议不通过 → 弹出驳议输入框 → 圣旨变红 + 回流中书
  → 中书重拟 → 再次提交门下 → 通过后继续流转
```

### 交互表现
- 点击「封驳」→ 弹出驳议理由框 → 提交后进度条回退至中书 → 敕令卡片边框变红 → 流转日志新增封驳记录

### 留痕规则
- 每次封驳永久记录，计入对应部门考核数据与勋章统计

---

## 4.2 密奏直达（特殊闭环）

```
六部/刑部发起密奏 → 直达天堂天子 → 天子批示 → 下发三省执行
```

### 权限规则
- 密奏内容仅发起方与天子可见，不经过三省常规流程

### Figma交互
- Hub浮窗选择「密奏」类型，提交后直接出现在天堂待办列表

---

## 4.3 四道硬校验规则

| 规则　　　　　 | 内容　　　　　　　　　　　　　　　　　 | Figma表现　　　　　　　　　　　| 系统行为　　　　　　 |
| ----------------| ----------------------------------------| --------------------------------| ----------------------|
| **无玺不生效** | 未加盖玉玺的指令，六部可拒绝执行　　　 | 「执行」按钮置灰，提示"待盖玺" | Agent拒绝启动任务　　|
| **无印不流转** | 缺少任一省官印的圣旨，下一节点自动拦截 | 进度条阻塞变红，无法点击下一步 | 流转引擎拦截　　　　 |
| **无牌不执行** | 未附带令牌的任务，六部Agent不启动资源　| 任务列表拒绝接收，Toast提示　　| 资源管理器拒绝分配　 |
| **覆奏闭环**　 | 任务必须回奏归档，否则占用令牌　　　　 | 超时铜牌变红，计入考核扣分　　 | 令牌不释放，超时预警 |
