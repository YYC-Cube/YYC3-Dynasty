## 六、交互规范

### 6.1 动效时长标准

| 动效类型 | 时长 | 缓动函数 | 适用场景 |
|----------|------|----------|----------|
| 微交互 | 100-200ms | ease-out | 按钮/hover/焦点 |
| 界面过渡 | 200-300ms | ease-in-out | 面板展开/收起 |
| 页面转场 | 300-500ms | ease-in-out | 路由切换 |
| 强调动效 | 500-800ms | cubic-bezier | 玉玺钤印/勋章解锁 |
| 循环动效 | 3s周期 | ease-in-out | 呼吸灯/加载动画 |

### 6.2 状态覆盖要求

每个可交互组件必须覆盖以下状态：

| 状态 | 视觉表现 | 示例 |
|------|----------|------|
| Default | 默认态 | 正常显示 |
| Hover | 悬浮态 | 微放大/边框变亮/透明度变化 |
| Active/Focus | 激活态 | 高亮边框/阴影加深 |
| Pressed | 按下态 | 缩小0.98/颜色加深 |
| Disabled | 禁用态 | 灰色50%透明度/不可点击 |
| Loading | 加载态 | 骨架屏/旋转图标 |
| Error | 错误态 | 红色边框/错误提示 |
| Empty | 空态 | 空插图/引导文案 |

### 6.3 响应式断点

| 断点 | 宽度 | 布局变化 |
|------|------|----------|
| Mobile | < 640px | 单列堆叠，隐藏次要信息 |
| Tablet | 640-1024px | 双列布局，侧边栏可折叠 |
| Desktop | 1024-1440px | 标准三栏布局 |
| Wide | > 1440px | 宽屏增强，显示更多信息 |
