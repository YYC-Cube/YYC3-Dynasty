## 一、产品概述

YYC³ Dynasty 是以中国古代"三省六部制"为思想根基的多智能体协同系统。UI 需体现 **古文化庄重感 + 科技轻盈感** 的双重气质。

### 核心理念

> **三省以治，六部以行。**

- 决策层 (皇帝) → 承启层 (太子) → 三省审议 (中书/门下/尚书) → 六部执行 (户礼兵刑工吏)
- 辅助层 (早朝) 定时播报
- 17 种中华文化勋章
- 13 王朝 × 26 Skills（双层技能体系：L1 文化层 + L2 职能层）

### 视觉关键词

`宣纸` `卷轴` `竹简` `墨色` `玉玺` `鎏金` `隶书` `印章红` `暗纹` `留白` `微光`

#### 4.3.1 双层技能模型架构

```
┌─────────────────────────────────────────────┐
│           双层技能协同模型                    │
│                                             │
│  L1 文化层 — 朝代 × 治世技能                  │
│  ├── 触发: 用户在时间轴选择/解锁朝代           │
│  ├── 效果: 为系统提供文化属性加成              │
│  │   例: 选择"唐"-> 桃李满天下(主动)激活       │
│  │       -> 所有 Agent 回复质量 +15%           │
│  │       -> 国色天香(被动) -> UI 金碧辉煌主题    │
│  └── 持续: 当前会话 / 直到切换朝代             │
│                                             │
│  L2 职能层 — 角色 × 业务技能                  │
│  ├── 触发: Agent 在流程中按条件自动触发         │
│  ├── 效果: 直接影响敕令流转结果                │
│  │   例: 门下省触发"执法如山"(被动)             │
│  │       -> 自动高亮违规项，建议封驳             │
│  │   例: 中书省触发"龙章凤藻"(主动)             │
│  │       -> 自动生成完整执行方案                │
│  └── 冷却: 政令值资源单位                      │
│                                             │
│  协同规则:                                   │
│  L1 朝代加成 x L2 技能基础效果 = 最终效果        │
│  例: "唐"朝代加成(x1.15) x 中书龙章凤藻         │
│    = 方案质量评分 x 1.15                       │
└─────────────────────────────────────────────┘
```

#### 4.3.2 技能卡片交互状态

| 状态 | 视觉表现 | 可执行操作 |
|------|---------|-----------|
| **未解锁** | 灰色卡片 + 锁图标 + "解锁条件: XXX" | 查看条件提示 |
| **已解锁·未激活** | 正常卡片 + "激活"开关(关闭态) | 点击激活 |
| **已解锁·已激活** | 金色边框呼吸光效 + "激活"开关(开启态) | 点击停用 / 展开详情 |
| **冷却中** | 卡片半透明 + 倒计时环 + "冷却: 02:30" | 等待恢复 |
| **被动常驻** | 绿色左竖线标识 + "被动"标签 | 无法手动触发，自动生效 |

```typescript
    activeDynasty: string | null;    // 当前激活朝代
    activeSkills: string[];           // 已激活技能ID列表
    currentView: PageRoute;           // 当前页面路由
  };
  
  // 敕令管理
  edicts: {
    active: Edict[];                  // 进行中的敕令
    completed: Edict[];               // 已完成的敕令
    drafts: Edict[];                  // 草稿箱
  };
  
  // 技能系统
  skills: {
    unlocked: Skill[];                // 已解锁技能
    cooldowns: Record<string, number>; // 技能ID → 恢复时间戳
  };
  
  // Agent状态
  agents: {
    list: Agent[];
    statusMap: Record<string, AgentStatus>;
  };
  
  // 系统
  system: {
    health: SystemHealth;
    metrics: PerformanceMetrics;
    notifications: Notification[];
  };
}
```

### 6.3 主要数据流向

```
用户操作 (User Action)
  │
  ├── 朝代切换 → Context.activeDynasty变更
  │     ├── 广播 skill:dynasty-changed
  │     ├── L1技能自动激活/停用
  │     └── UI主题可能变更
  │
  ├── 技能激活 → Skills.cooldowns更新
  │     ├── 广播 skill:activated
  │     └── 相关敕令效果系数重算
  │
  ├── 发起敕令 → Edicts.active新增
  │     ├── 自动进入第1阶(draft)
  │     ├── 广播 edict:created
  │     └── 通知相关Agent
  │
  ├── 推进阶段 → Edict.stage变更
  │     ├── 校验权限+资源
  │     ├── 执行阶段逻辑(草拟/审议/派发...)
  │     ├── 广播 edict:stage-changed
  │     └── 可能触发技能效果
  │
  └── 封驳操作 → Edict回到proposed
        ├── 生成封驳记录
        ├── 广播 edict:rejected
        └── 通知中书省重拟
```

### 7.3 并发控制策略

#### 敕令编辑锁

同一敕令同时只允许一个用户/Agent编辑：

```typescript
// 乐观锁实现
async function updateEdict(
  edictId: string,
  updates: Partial<Edict>,
  expectedVersion: number
): Promise<Result<Edict>> {
  const current = await edictRepo.getById(edictId);
  
  if (current.version !== expectedVersion) {
    return { success: false, error: 'CONFLICT', message: '敕令已被其他人修改' };
  }
  
  return await edictRepo.update(edictId, {
    ...updates,
    version: expectedVersion + 1,
    updatedAt: new Date(),
  });
}
```

#### 技能冷却互斥

同一技能在冷却期间无法再次激活（前端UI禁用 + 后端校验双重保障）：

```typescript
function canActivateSkill(skill: Skill, now: Date): Result<void> {
  if (skill.status === 'cooling_down') {
    const remaining = skill.cooldownUntil.getTime() - now.getTime();
    return { 
      success: false, 
      error: 'COOLDOWN', 
      message: `技能冷却中，剩余 ${Math.ceil(remaining/1000)} 秒` 
    };
  }
  return { success: true };
}
```

### 8.1 错误处理策略

| 错误类型 | 用户提示 | 恢复方式 | 日志级别 |
|---------|---------|---------|---------|
| 网络超时 | "网络不稳定，正在重试..." | 自动重试(最多3次，指数退避) | WARN |
| 权限不足 | "您暂无此操作权限" | 引导申请/切换角色 | INFO |
| 参数校验失败 | 具体字段错误提示 | 用户修正后重试 | DEBUG |
| 服务端异常 | "操作失败，请稍后重试" | 手动重试 | ERROR |
| 并发冲突 | "内容已被更新，请确认" | 显示冲突内容供合并 | WARN |
| 技能冷却中 | "技能冷却中，剩余 XX 秒" | 等待或使用其他技能 | INFO |

### 8.2 性能优化清单
