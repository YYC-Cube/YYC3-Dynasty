## 目录定位

本目录为 **规划文档归档区**，存放 Dynasty Framework 的原始设计规范文件。这些文档是整个 `docs/` 文档体系的**源头依据**，所有结构化拆分的文档（`01-` ~ `06-`）均基于此处的源文件进行模块化重构。

### 归档原则

- **完整性保留**：保持原始文档的完整内容，不做删减或修改
- **版本冻结**：归档后不再更新，仅作为历史版本参考
- **可追溯性**：提供从拆分文档回溯到源文件的路径

## 使用指南

### 何时查看规划文档？

1. **需要了解全貌时**：当拆分后的文档无法满足整体理解需求时
2. **版本对比时**：需要确认原始设计与当前实现的差异时
3. **问题追溯时**：在拆分文档中发现不一致，需回溯原始定义时

```
初学者路径：

  本目录（规划文档） → 理解完整框架 → 按需查阅 01-06 分类文档
```
