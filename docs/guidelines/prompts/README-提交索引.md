## 标准格式说明

每个提示词文件均包含以下结构：

```
【角色定位】    → AI 执行身份（如：资深组件设计师）
【任务目标】    → 明确的交付物
【前置依赖】    → 执行前必须具备的条件
【输出目标】    → 具体的交付物清单
【验收标准】    → 【AI可达成】+ 【人工后处理】分层标注
【优先级】      → P0/P1/P2

--- 复制以下内容提交 ---（以下全部复制到 Figma AI 对话框）

TASK: ...
CONTEXT: ...
...
--- 复制截止 ---

【提交后人工处理】→ AI 生成后需手动完成的步骤
```

---

## 使用流程

1. **首次使用**: 从 `00-Design-Tokens.md` 开始，按顺序逐个提交
2. **跨会话续接**: 每个文件顶部含「上下文锚点」，新对话粘贴即可重建上下文
3. **响应式生成**: 先提交 Desktop 版，再用同一文件的 Mobile 段落提交移动版
4. **人工后处理**: 每次提交后，按文件末尾「提交后人工处理」完成组件封装
